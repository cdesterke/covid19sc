IMMUNE DECONVOLUTION

Christophe Desterke

10/29/2020

This tutorial explain immune-deconvolution process for a transcriptome dataset

Immune deconvolution of transcriptome is a bioinformatics pipeline which integrate the transcriptome dataset from an organ with the transcriptome of sorted immune cell subpopulations. This procedure helps explain the level of immune infiltration of an organ. This original procedure have been employed in the following publication https://pubmed.ncbi.nlm.nih.gov/33015591/

In this procedure, the COVID19 patient lung dataset (GSE147507) is used along with the data from immune subpopulations included in the dataset (GSE76340).

references of the datasets:

  • Blanco-Melo D, Nilsson-Payant BE, Liu WC, Uhl S et al. Imbalanced Host Response to SARS-CoV-2 Drives Development of COVID-19. Cell 2020 May 28;181(5):1036-1045.e9. PMID: 32416070

  • Pont MJ, Honders MW, Kremer AN, van Kooten C et al. Microarray Gene Expression Analysis to Evaluate Cell Type Specific Expression of Targets Relevant for Immunotherapy of Hematological Malignancies. PLoS One 2016;11(5):e0155165. PMID: 27171398

First step is to correct cross batch effect between datasets

In order to correct batch effect in transcriptome SVA bioconductor package will be used. To perform this normalization step, two files are necessary: a file comprising the transcriptome data matrix of the two datasets (this matrix file is named covidcross2.txt) and a metadata file comprising in particular the information of the batches and incidentally data corresponding to biological groups (this file is named metadata.txt).

These pre-process data could be download at the address: https://github.com/cdesterke/covid19sc/tree/master/immunedeconvolution_data
